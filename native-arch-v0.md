# 画面録 ネイティブ版 アーキテクチャ memo v0（道B＝売れる版）

> Web版（本リポ）＝無料導線・コンセプト実証。自動取得の「売れる版」は**別物のネイティブiOSアプリ**。
> 最大の制約を最初から設計に織り込む：**Screen Time の集計値はサンドボックス外に出せない**。

## 0. 結論（3行）
- 自動取得は可能。だが「数値を取り出して好きに使う」はできない。**計算も描画も拡張内で完結**させる前提で組む。
- ゆえに「サーバ同期・実数フレンドランキング・複数端末の正確な合算」は素直に作れない。**ローカル完結＋共有は画像/称号**が現実解。
- それはバグでなく **プライバシー100%という売り**に変換できる（Opal の売り文句そのもの）。

## 1. ターゲット / ポジショニング
- 既存（Opal / Jomo / one sec / ScreenZen）＝ブロッカー or 罪悪感メーター。
- 画面録ネイティブ＝「スクリーンタイム＝鍛えてるステータス、攻略はバランス」＝**反・煽りRPG**。ウェッジは明確で空いている。

## 2. 技術スタック
- **Swift / SwiftUI**。
- **FamilyControls**：認可（`AuthorizationCenter`）＋ `FamilyActivityPicker`。
- **DeviceActivity**：使用量監視＋**`DeviceActivityReport`**（集計の表示）。
- **ManagedSettings**：任意（“呪い解除のために制限を足す”等を作るなら）。
- 配布に **Family Controls entitlement**（Apple へ申請・個別承認。distribution 用は審査あり＝**律速**）。

## 3. データフロー（制約込み・ここが設計の肝）

```
[iOS] が Screen Time を集計
   ↓ 認可済みなら
[DeviceActivityReport 拡張（サンドボックス）]  ← データは“入ってくる”だけ
   ・iOSカテゴリ → 6系統XP を算出
   ・六角レーダー＋系統別バー＋調和＋状態異常を SwiftUI で描画
   ✗ 集計値を本体アプリへ渡せない（App Group / UserDefaults / 共有ファイル 不可）
[本体アプリ]
   ・拡張の View を“埋め込んで表示”するだけ。生数値は持てない。
```

設計判断：
- 「ステータス画面」＝ **Report 拡張の View そのもの**。Web版の `compute()` を Swift に移植してここに置く。
- “数値を持ちたい”機能（履歴・称号の永続・サーバ同期）は拡張内から**自動では外に出せない**。
  → (a) 拡張内で完結する範囲に絞る、または
  → (b) ユーザーが能動的に「今週を保存」した時だけ、拡張内描画を**画像化してユーザー主導で**共有/保存（＝数値ではなく絵が出ていく）。

## 4. 機能の取捨（制約に正直に）
- **できる**：自動でステータス画面／調和／状態異常／攻略ヒント（すべて拡張内）。プライバシー100%訴求。
- **避ける/難しい**：実数のサーバ同期、実数フレンドランキング、複数端末横断の正確合算。
- **共有**：六角形の**画像＋ビルド称号**（例「均衡の賢者 ・調和0.82」）。数値露出は最小。

## 5. MVP の切り方（コア価値だけ先に）
1. 認可フロー（FamilyControls authorization）。
2. **Report 拡張で 6系統XP算出＋hexagon描画**（Web版ロジックの移植）。
3. 本体＝拡張View埋め込み＋オンボーディングのみ。
→ ここまでで「**自動で六角形が出る**」＝コア体験が成立。ここを最初の山に。

## 6. マネタイズ
- サブスク（Opal等の相場感）。コア（自動ステータス）を無料、**称号/見せ方/カスタム目標バランス**を有料、等。
- 履歴系は §3 制約のため拡張内 or ユーザー主導保存で工夫。

## 7. 工数 / リスク
- **数週間〜**の本格ネイティブ。**Apple entitlement 審査が律速＆不確実**（落ちる可能性あり）。
- 非公式SDK（STKManager 等）でのデータ抽出は**脆く・審査グレー・破損リスク**＝非推奨。
- まず Web版でコンセプト/ビジュアル/写像を確定（＝de-risk）してから着手が合理的。← **今ここ**。

## 8. 着手するなら（順番）
1. Apple Developer で **Family Controls entitlement 申請**（律速だから先に出す）。
2. Web版の `compute()`（写像・調和・状態異常）を**純ロジックに切り出し**→ Swift 移植しやすく整理。
3. **Report 拡張内の SwiftUI hexagon プロトタイプ1枚**で「自動で六角形」を最短実証。
