# 画面録（がめんろく） / Screen Time RPG — concept v0

> スクリーンタイムを「浪費の罪悪感」ではなく「すでに鍛えているステータス」として読み替えるツール。
> #全人類UX改善 / Visionium・アイデア苗床。設計思想：Boolean→Float。

---

## 1. 何を解くか（問題）

iOS標準のスクリーンタイムは「今週7h32m ↑23%」と**大きな数字で殴ってくる**。
これはBoolean寄りの罪悪感（多い/少ない、良い/悪い）。やめたくなるだけで、何も育たない。

**読み替え**：画面の前で過ごした時間は、すでに何かの**XP**になっている。
問いは「無駄にしたか（Yes/No）」ではなく「**どんなステータスの偏りで character を組んでいるか**」。
攻略は時間を**ゼロにすること**でも**XPを最大化すること**でもなく、**バランス（調和）を整えること**。

---

## 2. 前提と v0 スコープ

- iOS Screen Time API は非公開 → ブラウザ単一HTMLでは**自動取得不可**。
- **v0 = 手入力MVP**：iOS設定アプリの週次レポート（カテゴリ別時間）を手で打ち込む。
- 完了条件：手入力 → ステータス算出 → Lv／系統バー／調和メーター表示の LIVE。
- 自動取得（Shortcuts経由のCSV書き出し等）は v1 以降の検討事項。

## 3. 写像設計

### 3.1 アプリカテゴリ → 6系統（ステータス傾向）

RPG STATUS の正準6系統（RPG Status DB 準拠）：
**PHI 哲学 / CRE 創作 / TEC 技術 / OUT 発信 / VIT 体力 / EXP 探究**

各カテゴリは6系統への**重みベクトル**（合計1.0、主系統に集中）を持つ。
`xp[系統] = Σ_カテゴリ ( 分_カテゴリ × 重み[カテゴリ][系統] )`（1分=1XP）。

| iOSカテゴリ | 主写像（重み） | 過剰→状態異常 | 閾値/週 |
|---|---|---|---|
| ソーシャル | OUT .8 / PHI .1 / EXP .1 | 呪い「無限スクロールの呪縛」 | 7h |
| エンタメ（動画・配信） | VIT .5 / EXP .3 / PHI .2 | 状態異常「惰性のとろみ」 | 12h |
| ゲーム | EXP .6 / CRE .3 / VIT .1 | 状態異常「没入のとりつかれ」 | 12h |
| 創作 | CRE 1.0 | —（高ソフトキャップ） | — |
| 仕事効率化・金融 | TEC .6 / OUT .4 | — | — |
| 開発ツール | TEC 1.0 | — | — |
| 情報・読書 | PHI .7 / EXP .3 | — | — |
| 教育 | EXP .8 / PHI .2 | — | — |
| 健康・フィットネス | VIT 1.0 | — | — |
| 旅行・ナビ | EXP .7 / VIT .3 | — | — |
| 買い物・食 | VIT .2（ほぼ中立） | 状態異常「浪費のうずき」 | 6h |
| その他・ユーティリティ | 中立（XPなし） | — | — |

### 3.2 個別アプリ → 技・スキル

よく開いたアプリ名を打つと、**習得した技**として鑑定される（フレーバー。XPは二重計上しない）。
例：GitHub→「鍛冶の手」(TEC) / Clip Studio→「彩色の極み」(CRE) / Duolingo→「語学の心得」(EXP) /
X→「拡声の術」(OUT・※過ぎれば呪い) / Kindle→「読みの灯」(PHI) / ヘルスケア→「身体の声」(VIT)。
辞書未収録は「未鑑定の技」。

### 3.3 過剰 → 状態異常（Boolean→Float の核）

「SNS=悪」ではない。少量のSNSは健全な OUT（つながり）。
**閾値を超えた分だけ**連続値で状態異常が立ち上がる：
`強度 = clamp((時間 − 閾値) / 閾値, 0, 1)`（2倍で最大）。
強度>0 の系統は warn 色に転び、状態異常カードと**解呪ヒント**が出る。
- 全体 > 56h/週（≒8h/日）→ 「過暴露（オーバーエクスポージャー）」をやさしく通知。
- 単一系統シェア > 0.55 → 「偏り」を通知。

## 4. 算出ロジック（数値で煽らない設計）

- **調和（Harmony）= 主役 Float 0.00–1.00**：6系統XPシェアの正規化シャノンエントロピー `H/ln6`。
  全系統を均等に使うほど 1.0 に近づく。**これが報酬軸**。
- **クリア判定**：調和 ≥ 0.75 かつ 状態異常なし → 「バランス・クリア」。
- **Lv = ⌊総XP / 847⌋**（既存グラマー流用）。ただし**小さく副次表示**＝「今週の鍛錬量」。
  「多ければいいわけじゃない」と明記。攻略判定には**使わない**（時間の総量を煽らないため）。
- **攻略ヒント**：原則「**多い系統を削れ**」ではなく「**手つかずの系統を足せ**」と促す
  （育てる方向＝やさしいBoolean→Float）。ただし状態異常（呪い）だけは「解呪＝過剰を緩める」を提案。

## 5. 描画グラマー（流用元）

- **六角レーダー**（6軸・手書きSVG）：正六角形に近いほどバランス良。gold ポリゴン、状態異常軸は warn。
- **系統別バー**：miharashi.html の `bar(label,value,max)` グラマー（`.rpg-grid` ＝ label / bar `<i width%>` / value / pct）を流用。
- **配色規律**：colors_and_type.css 準拠。暖色は gold 単一、状態異常のみ warn。rainbow/ネオン禁止。
  系統の識別は**ラベル横の小さな減彩ドット**のみ（データ面は gold で統一）。
